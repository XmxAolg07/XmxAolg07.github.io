汇编笔记-1

一、6 种常用Debug指令

指令 全称 作用说明 GDB 示例
int 3 Interrupt 3 软件断点。程序执行到此会触发调试器暂停。常用于手动插入断点。 无需输入，CPU遇到即停。
nop No Operation 空操作。占用一个字节 (0x90)，什么都不做。常用于对齐代码或预留补丁位置。 x/10i $rip (查看后续指令)
ud2 Undefined Instruction 未定义指令。强制产生 SIGILL 非法指令异常，调试器会捕获。用于标记不该执行的代码路径。 run 后自动停在异常处。
pushf / pushfq Push Flags 保存标志寄存器。将 RFLAGS/EFLAGS 压栈，调试时用于查看条件码（如 ZF, CF）。 print $eflags
cpuid CPU Identification 获取CPU信息。常用于检测是否支持特定指令集（如 AVX），调试时用来确认环境。 info registers eax
hlt Halt 停机。使CPU进入低功耗暂停状态，直到收到中断。在用户态程序中使用会触发 SIGSEGV 或异常。 通常用于内核调试，用户态极少用。

三、寄存器

CPU是由运算器，控制器，寄存器等构成

  • 运算器进行信息处理
  • 寄存器进行信息储存
  • 控制器控制器件进行工作

  • 四个通用寄存器

64位 低32位 低16位 低8位 特殊用途 / 备注
RAX EAX AX AL 累加器。mul/div 隐式使用；函数返回值通常存于此。
RBX EBX BX BL 基址寄存器。被调用者保存（Callee-saved），函数内用需先压栈。
RCX ECX CX CL 计数器。loop 指令；shl/shr 移位位数非立即数时存于此。
RDX EDX DX DL 数据寄存器。mul 乘法结果高64位存此；div 除法余数存此；系统调用第3参数。

指令 作用 实例 执行后效果
