汇编笔记-2

一、核心概念：物理地址的计算

在实模式下，CPU 通过 段地址 和 偏移地址 组合生成 20 位的 物理地址。

术语 英文 作用 计算规则
DS Data Segment 数据段寄存器，指向数据存放的内存段基址 物理地址 = DS × 16 + 偏移地址
CS Code Segment 代码段寄存器，指向指令存放的内存段基址 物理地址 = CS × 16 + IP
SS Stack Segment 栈段寄存器，指向栈空间的内存段基址 物理地址 = SS × 16 + SP
ES Extra Segment 附加段寄存器，常用于字符串操作的目标地址 物理地址 = ES × 16 + DI

二、CS:IP 与 JMP 指令（程序跳转）

CS (代码段) 和 IP (指令指针) 组合决定了 CPU 下一条要执行哪里的指令 。

1. 寄存器状态

寄存器对 作用 说明
CS:IP 指向当前正在执行的指令 不可直接通过 mov 修改，需通过跳转指令修改

2. jmp 指令实例表

指令格式 实例 执行效果
段内短跳转 jmp short label IP = IP + 位移量 (范围 -128~127 字节)
段内近跳转 jmp near label``jmp ax IP = 新偏移地址 (CS 不变)
段间远跳转 jmp 0x1000:0x0000 CS = 0x1000, IP = 0x0000 (同时修改段和偏移)

三、SS:SP 与栈操作（PUSH / POP）

栈是向下增长的（高地址向低地址），SS 指向栈段，SP 指向栈顶。

1. 寄存器定义

寄存器对 作用 栈顶物理地址计算
SS:SP 指向栈顶元素 物理地址 =SS × 16 + SP

2. 入栈与出栈指令

指令 操作流程 SP 变化 等效动作 (仅示意逻辑)
push ax 1. SP = SP - 2``2. 将 AX 存入 [SS:SP] 减小 2 sub sp, 2 + mov [ss:sp], ax
pop ax 1. 从 [SS:SP] 取值给 AX``2. SP = SP + 2 增加 2 mov ax, [ss:sp] + add sp, 2
pushf 将标志寄存器 FLAGS 压栈 减小 2 保存所有状态标志位
popf 从栈恢复 FLAGS 增加 2 恢复状态标志位

四、综合调试示例

在 Debug 中查看这些寄存器组合：

命令 作用 示例输出解读
R 查看所有寄存器 DS=13A7 CS=13A8 SS=13A7 SP=00FE
D DS:0 查看数据段开头 显示 13A7:0000 处内存数据
U CS:IP 反汇编当前指令 显示接下来要执行的代码
T 单步执行 观察 push 后 SP 减 2；pop 后 SP 加 2

常见的段寄存器隐含使用

  • 访问数据默认用 DS:[偏移]，如 mov ax, [bx]
  • 栈操作默认用 SS:[SP]，如 push / pop
  • 指令读取默认用 CS:[IP]
  • 字符串写操作 (stosb) 强制使用 ES:[DI]

五、x86 常用寻址方式详解

说明：EA 为有效地址（Effective Address），SA 为段地址（Segment Address）。
默认段约定：除 BP 默认配合 SS 段外，其余寄存器默认配合 DS 段。

1. 直接寻址

格式 EA 计算 默认段 示例
[idata] EA = idata DS mov ax, [1234h]

2. 寄存器间接寻址

格式 EA 计算 默认段 示例
[bx] EA = (BX) DS mov ax, [bx]
[si] EA = (SI) DS mov ax, [si]
[di] EA = (DI) DS mov ax, [di]
[bp] EA = (BP) SS mov ax, [bp]

3. 寄存器相对寻址

格式 EA 计算 默认段 等效写法 常见用途
[bx + idata] (BX) + idata DS [bx].idata 结构体成员
[si + idata] (SI) + idata DS idata[si] 一维数组
[di + idata] (DI) + idata DS idata[di] 一维数组
[bp + idata] (BP) + idata SS [bp].idata 栈内参数访问

4. 基址变址寻址

格式 EA 计算 默认段 等效写法 常见用途
[bx + si] (BX) + (SI) DS [bx][si] 二维数组
[bx + di] (BX) + (DI) DS [bx][di] 二维数组
[bp + si] (BP) + (SI) SS [bp][si] 栈内二维访问
[bp + di] (BP) + (DI) SS [bp][di] 栈内二维访问

5. 相对基址变址寻址

格式 EA 计算 默认段
[bx + si + idata] (BX) + (SI) + idata DS
[bx + di + idata] (BX) + (DI) + idata DS
[bp + si + idata] (BP) + (SI) + idata SS
[bp + di + idata] (BP) + (DI) + idata SS

六、标志寄存器 FLAGS 各标志位详解

缩写 全称 中文名称 置 1 条件 主要用途
ZF Zero Flag 零标志 运算结果 = 0 je / jz 条件跳转
CF Carry Flag 进位标志 最高位产生进位/借位 jc / jnc，无符号数溢出判断
AF Auxiliary Carry Flag 辅助进位标志 第 3 位向第 4 位进位 BCD 码调整指令 (daa, aaa)
PF Parity Flag 奇偶标志 结果低 8 位中 1 的个数为偶数 串行通信数据校验
DF Direction Flag 方向标志 DF=1 时地址递减 串操作方向控制 (cld/std)
IF Interrupt Flag 中断标志 IF=1 时允许响应可屏蔽中断 sti(开中断) / cli(关中断)
TF Trap Flag 陷阱标志 TF=1 时 CPU 单步执行 调试器单步调试

标志位快速记忆口诀

  • ZF：结果是 Zero（零）
  • CF：最高位有 Carry（进位）
  • PF：1 的个数是 Parity（偶数）
  • DF：地址 Decrease（减少）还是 Increase
  • IF：Interrupt（中断）是否允许
  • TF：Trap（陷阱/单步）

七、寻址方式与段寄存器对应关系速查表

寻址组合 默认段寄存器 可否超越前缀 典型应用场景
[bx] / [si] / [di] DS 可改为 ES/SS/CS 访问数据段
